## Project Overview

Uncommit is a **pure native macOS menu bar app** written entirely in Swift 6.0. There are no Node.js, web, or third-party dependencies. The UI framework is SwiftUI, the minimum macOS target is 14.0 (Sonoma), and the project file (`.xcodeproj`) is generated by **XcodeGen** from `project.yml`.

---

## Build Commands

### Prerequisites
- Xcode 16.0+
- XcodeGen (`brew install xcodegen`)

### Generate the Xcode project (required after modifying `project.yml`)
```bash
xcodegen generate
```

### Open in Xcode
```bash
open Uncommit.xcodeproj
```

### CLI build (Release)
```bash
xcodebuild -project Uncommit.xcodeproj \
           -scheme Uncommit \
           -configuration Release \
           build
```

### CLI build (Debug)
```bash
xcodebuild -project Uncommit.xcodeproj \
           -scheme Uncommit \
           -configuration Debug \
           build
```

## Lint / Format

There is currently **no SwiftLint, SwiftFormat, or CI pipeline** configured. Follow the style conventions in this file and validate correctness by building cleanly with `xcodebuild`.

## Tests

There is currently **no test target**. To add tests, add a `UncommitTests` target to `project.yml`, re-run `xcodegen generate`, and write `XCTest`-based unit tests. To run a single test from the CLI once a test target exists:

```bash
xcodebuild test \
  -project Uncommit.xcodeproj \
  -scheme UncommitTests \
  -only-testing:UncommitTests/SuiteClassName/testMethodName
```

## Code Style Guidelines

### Swift Version & Compiler Settings
- **Swift 6.0** (`SWIFT_VERSION = "6.0"` in `project.yml`)
- **`SWIFT_STRICT_CONCURRENCY: complete`** ŌĆö all data-race safety is enforced at compile time; treat every concurrency warning as an error
- **`MACOSX_DEPLOYMENT_TARGET: "14.0"`** ŌĆö only macOS 14+ APIs are available

### File Organization
- One primary type per file; file name matches the type name (PascalCase, `.swift` extension)
- Small private sub-views defined in the same file as their parent view are acceptable
- Sections within a file are delimited with `// MARK: - SectionName` comments

### Naming Conventions

| Category | Convention | Examples |
|---|---|---|
| Structs (models) | `PascalCase` noun | `GitRepository`, `GitRepoStatus`, `WatchedFolder` |
| Classes | `PascalCase` noun, always `final` | `AppViewModel`, `RepoMonitor`, `PersistenceService` |
| Namespace enums (no cases) | `PascalCase` noun | `GitService`, `ShellExecutor`, `EditorHelper`, `AppConstants` |
| Value enums | `PascalCase` noun | `ShellError`, `RepoHealthLevel` |
| SwiftUI Views | `PascalCase` + `View` suffix | `RepoRowView`, `StatusIndicatorView`, `EmptyStateView` |
| ViewModels | `PascalCase` + `ViewModel` suffix | `AppViewModel` |
| Services | `PascalCase` + `Service` suffix | `PersistenceService`, `RepoDiscoveryService` |
| Properties & methods | `camelCase` | `isRefreshing`, `watchedFolders`, `fullStatus(at:)` |
| Boolean properties | `is`/`has` prefix | `isRefreshing`, `hasUnpulledChanges`, `isClean` |
| Logger constant | `private let logger` (file-private, module-level) | `private let logger = Logger(subsystem: "com.uncommit.app", category: "ClassName")` |

### Imports
- `Foundation` ŌĆö models, services, utilities (avoid in pure SwiftUI view files)
- `SwiftUI` ŌĆö all views and `AppViewModel`
- `AppKit` ŌĆö only when using `NSWorkspace`, `NSOpenPanel`, `NSImage`, etc.
- `os` ŌĆö only in files that use `Logger`
- Imports go at the top of the file; order roughly by weight: `Foundation` ŌåÆ `SwiftUI` ŌåÆ `AppKit` ŌåÆ `os`

### Access Control
- Default to the most restrictive access level (`private` > `internal` > `public`)
- Use `private(set)` for properties that are publicly readable but internally writable
- Mark all class types `final` unless inheritance is explicitly required

### State Management (MVVM + `@Observable`)
- `AppViewModel` is the **single source of truth**; do not create additional view models
- Mark it `@Observable @MainActor final class` ŌĆö never use `ObservableObject`/`@Published`
- Views receive the model via `@Environment(AppViewModel.self)` injected from `UncommitApp`
- Views call `viewModel.method()` for sync work and `Task { await viewModel.asyncMethod() }` for async work
- Every mutation that must survive restarts ends with `saveConfiguration()`

### Concurrency
- `AppViewModel` and `RepoMonitor` are `@MainActor` ŌĆö all state mutation happens on the main thread
- Background work uses `async let` (parallel tasks) and `withTaskGroup` (sliding-window concurrency)
- Never call `Process.waitUntilExit()`; bridge `Foundation.Process` to async/await via `withCheckedThrowingContinuation` + `terminationHandler` (see `ShellExecutor.swift`)
- Use `[weak self]` in `@Sendable` closures that capture `self` to avoid retain cycles
- Guard against double-initialization from SwiftUI re-renders with an idempotency flag (`guard !hasStarted`)

### Error Handling
Use the pattern appropriate to context:

1. **`Result<T, Error>`** at service boundaries (e.g., `GitService ŌåÆ RepoMonitor`) so errors don't escape across `TaskGroup`
2. **`throws` + `try/catch`** at the shell level; callers make policy decisions (silently return a default, log, or propagate)
3. **`@MainActor @Sendable` callbacks** (`onStatusUpdate`, `onError`) for monitor-to-viewmodel error propagation
4. **Typed `enum ŌĆ” : Error, LocalizedError`** with a meaningful `errorDescription` (see `ShellError`)
5. **Silent fallback / graceful degradation** with `try?` for non-critical decoding (e.g., `PersistenceService` returns `AppConfiguration()` on failure); never crash on recoverable errors
6. Surface errors in the UI as **inline red `Text`** (caption font) in the relevant row ŌĆö never as modal alerts

### Persistence
- All persistence goes through `PersistenceService` ŌåÆ `UserDefaults` + `JSONEncoder`/`JSONDecoder`
- No Core Data, no SQLite, no file-based storage

### Project Configuration
- Do **not** commit `Uncommit.xcodeproj/`; it is regenerated by XcodeGen
- Edit `project.yml` to add new source files, targets, or build settings, then re-run `xcodegen generate`
- App sandbox is disabled (`ENABLE_APP_SANDBOX: NO`) to allow running git subprocesses and opening Terminal
- The app is menu-bar-only (`LSUIElement = true` in `Info.plist`); there is no Dock icon or main window
